Section 2

How Do You Talk to a Computer?

Your phone understands "Hey Siri, play something chill" — but it doesn't understand you the way a friend would. There's a big difference between how humans communicate and how computers actually work.

🗣️
Natural vs. Formal Language Natural language (English, Spanish) is flexible and messy. Formal language (Python, Java) is exact and unambiguous. Computers can only work with formal language.
Natural Language Formal Language
Examples English, Spanish, emoji Python, Java, HTML
Rules Flexible, context-dependent Exact, unambiguous
Errors Humans usually figure it out Computer stops completely
Ambiguity Common and acceptable Not allowed

See the difference

These two instructions mean the same thing to humans — but only one works for a computer.

Version A — Natural Language

"Hey, can you send me that homework thing? You know the one."

A human figures it out from context. A computer has no idea — what thing? Which homework?

Version B — Formal Language

sprite.move_right(100)

Exactly one interpretation. Move the sprite right by 100 pixels. No guessing needed.

🐍
Python — the language you'll use in this course Python is one of the most widely used programming languages in the world. It's used in AI research, data science, web development, and cybersecurity — and it's the same language professionals use. You'll write in Python through Codesters.

Section 3

What's an Algorithm?

Every recommendation you've gotten from a streaming service, every search result, every weirdly specific ad — those are all outputs of algorithms someone wrote.

📋
Algorithm A set of precise steps for solving a problem or completing a task. The steps must be in the right order, and nothing can be skipped.

How TikTok Decides What's Next

It's not random. It's not a human picking for you. An algorithm runs constantly, weighing dozens of signals. Step through how it works.

1

Collect your behavior

How long did you watch the last video? Did you like it, share it, or skip it immediately? Every action is recorded as data.

2

Look at your history

What have you liked, commented on, or rewatched before? The algorithm builds a profile of your preferences over time.

3

Find similar users

Who else watched what you watch? If millions of users with your taste also loved a certain video, you're likely to like it too.

4

Factor in context

What time is it? What device are you on? What's trending right now? Context signals also influence what gets shown.

5

Score and rank candidates

The algorithm scores dozens of potential videos against all these signals and picks the highest-ranked one.

6

Show video → repeat

Your reaction to this video feeds right back into Step 1. The loop never stops — and it gets better with every cycle. ✓

✅
Algorithm complete. Every step in order, every signal considered — that's why the feed feels like it knows you.
🤔
Think about it Who benefits when TikTok's algorithm gets really good at predicting what you'll watch? The user? The creators? TikTok? Advertisers? All of them — but in different ways.

Key Vocabulary

algorithm
algorithm — a precise set of steps for solving a problem or completing a task. Steps must be in the correct order to work.
Section 4

When Things Go Wrong — Debugging

Your code will not work the first time. This is not a sign you're bad at coding. It's just how coding works. The skill is learning to find and fix the problem.

🐛
A brief history of "bug" The term dates to 1947, when a literal moth got stuck in a computer relay and caused it to malfunction. The engineer taped it into the logbook and wrote "first actual case of bug being found." The name stuck.
❌

Syntax Error

You broke a rule of the language — a missing quotation mark, a misspelled command, a parenthesis in the wrong place. The computer can't even read the instruction. The program crashes immediately.

⚠️

Logic Error

The code runs fine, but it does the wrong thing. Your instructions were precise — but they were wrong. Like giving someone directions that are technically correct but send them to the wrong place.

Identify the Bug Type

For each scenario below, decide: is it a Syntax Error or a Logic Error? Select your answer to check.

Bug A — A program is supposed to move a sprite to the right, but it moves left instead. The code runs fine, just does the wrong thing.
The program runs — so it's not a syntax error. The code told the sprite to go the wrong direction. That's a logic error: the instructions were precise but wrong.
Bug B — sprite.say("Hello there!) — the program crashes immediately when you run it.
The missing closing quote mark (") breaks Python's syntax rules. The computer can't even parse the instruction — it crashes immediately. That's a syntax error.
Bug C — An alarm app runs perfectly, but the alarm goes off one hour before the time the user set.
The app runs without crashing — so no syntax error. But the time calculation is wrong (probably a timezone or subtraction mistake). The code is precise, but the logic is off. Logic error.

Section 5

Hardware & Input — The Physical Side

Software runs on physical machines. Without input, a program just runs the same thing on a loop forever. Input is what makes software respond to you.

🖥️

Hardware

The physical components of a computer system — your phone, laptop, keyboard, camera. Hardware is what software runs on.

💾

Software

Programs and instructions that run on hardware. The apps on your phone, the operating system, the code you write — all software.

Input Devices

Some hardware is specifically designed to bring information in to a computer. Click a device to see how it works.

📱

📱 Touchscreen

Takes in taps and swipes directly on the screen. How your phone unlocks, how you scroll through Instagram, how a McDonald's kiosk takes your order — all touchscreen input.

🎤 Microphone

Takes in sound. When you say "Hey Siri" or "OK Google," your voice is captured as audio data, converted to text, and matched against patterns to figure out what to do.

📷 Camera

Takes in images. Face ID uses your camera to capture a 3D map of your face and compare it against stored data. If it matches, your phone unlocks — a real-time AI decision from a physical input.

🌡️ Sensor

Takes in physical measurements — temperature, light level, speed, location. Sensors in a self-driving car read the road, other vehicles, and obstacles constantly in real time, feeding data into algorithms that make driving decisions.

⌨️ Keyboard

Takes in typed characters and shortcuts. One of the most common input devices — translating physical key presses into digital characters the computer can process.
