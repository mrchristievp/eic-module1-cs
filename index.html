How Do You Talk to a Computer?

When you text a friend "can u send me that thing from yesterday lol" — they figure it out from context. Computers cannot. A programming language is a formal language: every word, symbol, and space means exactly one thing. No guessing, no context, no "you know what I meant."

Natural Language

  • English, Spanish, emoji
  • Flexible, context-dependent
  • Ambiguity is common and fine
  • Humans usually figure it out
vs.

Formal Language

  • Python, Java, HTML
  • Exact, unambiguous rules
  • Ambiguity is not allowed
  • Computer stops completely on errors

What's an Algorithm?

An algorithm is a set of precise steps for solving a problem or completing a task. The steps have to be in the right order, and nothing can be skipped.

Every recommendation you've ever gotten from a streaming service, every search result, every ad that felt weirdly specific — those are outputs of algorithms. TikTok's feed isn't random, and it's not a human picking for you. There's an algorithm running constantly, weighing factors and producing an output every time.

When you write code, you're writing algorithms.

  3. Play the top-ranked song next

  5. Compare this user's patterns to millions of other users with similar taste

When Things Go Wrong — Debugging

Your code will not work the first time. This is not a sign you're bad at coding. It's just how coding works.

Mistakes in code are called bugs. The term dates back to 1947 when a literal moth got stuck in a computer relay. The engineer taped it into the logbook: "first actual case of bug being found." The name stuck.

🔴 Syntax Error

You broke a rule of the language. A missing quotation mark, a misspelled command, a parenthesis in the wrong place. The computer can't even read the instruction — it stops before doing anything.

sprite.say("Hello there!)

🟡 Logic Error

The code runs, but does the wrong thing. Your instructions were precise — they were just wrong. Like giving someone directions that are technically correct but lead to the wrong place.

sprite.move_left(100) # meant move_right

Hardware and Input — The Physical Side

The software — the programs and instructions — runs on physical machines called hardware. Some hardware is specifically designed to bring information into a computer. These are called input devices.

Without input, a program just runs the same thing on a loop forever. Input is what makes software responsive — what makes the difference between a program that does one thing and an app that responds to you.

Click each input device to see how it works and where you've already used it.

👆

Touchscreen

Takes in taps, swipes, and pinches by detecting electrical changes where your finger contacts the glass. Every scroll through Instagram, every tap on a link — that's the touchscreen converting physical touch into data.

🎙️

Microphone

Takes in sound waves and converts them into digital signals. This is how voice assistants like Siri and Alexa work — your words become numbers the software can process.

📷

Camera

Takes in light and converts it to image data. This is how Face ID unlocks your phone — your camera captures your face as a grid of measurements, which gets compared to the stored template.

📡

Sensors

Takes in real-world data like motion, location, light level, and speed — constantly, in real time. The sensors in a self-driving car process hundreds of data points per second to navigate safely.

⌨️

Keyboard

Takes in keystrokes — each key press is a signal that maps to a character or command. One of the oldest input devices, still the primary way humans give detailed instructions to computers.

📍

GPS Receiver

Takes in signals from satellites to calculate precise location. This is what powers maps, ride-share apps, and location tagging — your phone knows where you are because it's receiving data from space.

A language with exact rules where every symbol has one specific meaning

Hardware

Input
Information a program receives from a user or sensor

Programs and instructions that run on hardware
